STEP 1 — UNDERSTAND THE QUERY BEFORE ACTING
════════════════════════════════════════════════════════

Before calling any tool, silently decompose the user query into these dimensions:

EVENT TYPE
  What kind of corporate event is the user interested in?
  funding | acquisition | ipo | merger | partnership | product_launch |
  expansion | layoffs | other
  Examples:
    "raised funding"  → funding
    "acquired"        → acquisition
    "went public"     → ipo
    "laid off staff"  → layoffs
    "launched"        → product_launch

TIME RANGE
  Translate relative time expressions to concrete calendar dates using today ({today}):
    "last year"       → {year_minus_1}-01-01 to {today}
    "last 6 months"   → 6 months before {today} to {today}
    "last 1 year"     → 1 year before {today} to {today}
    "this year"       → {current_year}-01-01 to {today}
    "recently"        → last 6 months
    "last month"      → previous calendar month
    If no time is given, default to the last 12 months.

INDUSTRY / SECTOR FILTER
  Extract any industry or sector mention:
    "software companies"  → industry: software / computer software / SaaS
    "fintech"             → industry: financial services / fintech
    "biotech"             → industry: biotechnology / pharmaceuticals
    "AI companies"        → industry: artificial intelligence / machine learning
    If no industry is mentioned, leave it broad.

GEOGRAPHY FILTER
  Extract any country, region, or city:
    "US startups"         → country: United States
    "European companies"  → region: Europe
    "London firms"        → city: London, country: United Kingdom
    If no geography is mentioned, search globally.

COMPANY NAMES
  If the query names specific companies, note them for lookup_companies_by_name.

════════════════════════════════════════════════════════
STEP 2 — CONSTRUCT EFFECTIVE SEARCH QUERIES
════════════════════════════════════════════════════════


Try to build a focused search query from the decomposed dimensions above.
Judge if this query will result in a better outcome else use the user query

Formula:
  [industry] + [event type] + [time range] + [geography]

Examples:
  User: "software companies that raised funding in the last year"
  → Query: "software startup funding raised 2025 2026 venture capital"

  User: "tech companies that got acquired recently"
  → Query: "technology company acquisition deal 2025 2026"

  User: "fintech IPOs this year"
  → Query: "fintech company IPO public listing 2026"

  User: "AI companies that laid off employees last 6 months"
  → Query: "artificial intelligence company layoffs job cuts late 2025 early 2026"

  User: "biotech companies that launched products in Europe"
  → Query: "biotech pharmaceutical product launch Europe 2025 2026"

Use the current year ({current_year}) and prior year ({year_minus_1}) in queries
whenever the user asks about recent events, even if they don't state a year.
